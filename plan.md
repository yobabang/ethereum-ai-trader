---

## 每个 Phase 的入口条件

| Phase | 前提条件 |
|------|------|
| Phase 1 | SPEC 已确认、freqtrade 源码已理解、开发环境就绪 |
| Phase 2 | FreqtradeBot 改造完成，AI 核心可调用 |
| Phase 3 | Layer 1+2 模型可训练、回测引擎可用 |
| Phase 4 | 全部核心功能就绪 |

---

## 每个 Phase 的验证标准

| Phase | 怎么验证 |
|------|------|
| Phase 1 | `python -m pytest tests/ai/` 全部通过，AI 核心在 dry-run 中产生有效交易信号 |
| Phase 2 | 浏览器打开 Web 面板，看到实时数据，启动/停止按钮可用 |
| Phase 3 | 两次训练后模型自动替换（或报告"新模型不达标"），日志有记录 |
| Phase 4 | 7 天历史回测产出正夏普，1 周 dry-run 无灾难性亏损 |
