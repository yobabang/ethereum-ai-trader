## Day 1: 代码审计 + 安全审查

### 1.1 代码审查 (code-reviewer agent)
- 审查全部 12 个 AI 模块（~5000 行）
- 检查维度: 正确性、可读性、架构、安全、性能
- 重点关注: DecisionArbitrator 安全规则是否完备
- 输出: 审查报告 + 问题清单

### 1.2 安全审计 (security-auditor agent)
- API 密钥管理审查
- 交易权限边界检查
- 异常处理完备性
- 资金安全风险评估
- 输出: 安全审计报告

### 验收标准
- [ ] 代码审查报告完成，Critical 问题 = 0
- [ ] 安全审计报告完成，高风险项 = 0

## Day 2: 数据扩充 + 模型重训练

### 2.1 数据下载
- 下载 BTC/ETH 完整 2025-2026 年数据（≥2000 根 K 线/币）
- 覆盖: 2024 年牛市、2025 年震荡、2026 年回调
- 数据来源: OKX 永续合约 4h K 线

### 2.2 模型重训练
- 用扩大后的数据集重新训练 RegimeClassifier + DirectionPredictor
- 时间序列交叉验证 (80/20 split)
- 记录训练指标基线

### 2.3 测试验证 (test-engineer agent)
- 数据完整性测试
- 特征质量验证
- 模型过拟合检测（前向验证）

### 验收标准
- [ ] 每个币种 ≥ 2000 根 K 线
- [ ] 训练集准确率 > 0.95（分类器）
- [ ] 前向验证无显著性能衰减

## Day 3: 回测验证 + 参数优化

### 3.1 全市场状态回测
- 牛市区间: 2024-01 至 2024-03（BTC +80%）
- 熊市区间: 2025-05 至 2025-09（BTC -40%）
- 震荡区间: 2024-06 至 2024-10（BTC 横盘）
- 极端行情: 2026-02-03（单日 -15% 闪崩）

### 3.2 参数网格搜索
- 杠杆: 2x, 3x, 5x
- 仓位: 10%, 20%, 30%, 50%
- 止损: 3%, 5%, 8%, 10%
- 置信度阈值: 0.50, 0.55, 0.60, 0.65
- 搜索目标: 最大化夏普比率，约束最大回撤 < 20%

### 3.3 优化配置输出
- 每个币种的最优参数组合
- 不同市场状态的最优参数

### 验收标准
- [ ] 4 种市场状态全部完成回测
- [ ] 最优参数组合使 MaxDD < 20%
- [ ] Sharpe > 1.0 on optimal config

## Day 4: 风险评估 + 边界测试

### 4.1 蒙特卡洛模拟
- 1000 次随机交易序列模拟
- 95% 置信区间的最大回撤
- 破产概率评估

### 4.2 极端行情测试
- 2020-03-12 式暴跌（-50% 单日）
- 2021-05-19 式闪崩（-30%）
- 插针行情（-20% 瞬间反弹）
- 长时间低波动（考验耐心和资金费率消耗）

### 4.3 交易所故障模拟
- API 超时处理
- 断线重连
- 错误订单处理

### 验收标准
- [ ] 破产概率 < 5%
- [ ] 极端行情下系统不崩溃
- [ ] 95% VaR < 15%
