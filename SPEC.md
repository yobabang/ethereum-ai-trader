# SPEC: 以太 / ETHEREUM — AI 自主合约交易代理

> **版本**: 0.1.0 | **日期**: 2026-06-28 | **状态**: 已确认
>
> 基于 Freqtrade 深度改造的纯 AI 驱动加密货币永续合约自动交易系统

---

## 一、产品定义

### 1.1 一句话描述

一个你只需要点击"启动"的 AI 交易机器人——它自己看市场、自己做决定、自己下单、自己优化，你只看结果。

### 1.2 核心原则

| 原则 | 含义 |
|------|------|
| **零决策** | 用户不需要选择策略、调参数、判断行情。只负责启动/停止 |
| **纯 AI** | 所有交易决策由一个 AI 模型端到端产出 |
| **自进化** | 每笔交易结果自动反馈到模型，持续优化 |
| **只做 BTC/ETH** | 永续合约，USDT 本位，流动性最好、庄家操控概率最低 |
| **安全第一** | 宁可错过机会也不冒超出承受范围的风险 |

### 1.3 用户画像

- **一个人使用**，不是 SaaS
- 有基本的加密货币交易知识，但不想盯盘
- 目标是让钱自己工作，而不是手动交易

## 三、AI 决策核心

这是整个系统的大脑。一个 4 层流水线，每 4 小时运行一次：

### 3.1 市场状态分类器（Regime Classifier）

**做什么**：判断当前市场处于什么状态，决定是否适合交易。

```
输入特征（实时计算）：
  ├── 波动率（最近 24h ATR / 价格）
  ├── 成交量轮廓（当前量 / 20周期均量）
  ├── 资金费率（正=多头过热，负=空头过热）
  ├── 趋势强度（ADX）
  ├── 订单簿不平衡度（买卖挂单比）
  └── 最近 24h 价格变化率

输出（分类）：
  ├── TRENDING_STRONG  → 强趋势，适合顺势交易
  ├── TRENDING_WEAK    → 弱趋势，可交易但降仓位
  ├── RANGING_TIGHT    → 窄幅震荡，不适合交易
  ├── RANGING_WIDE     → 宽幅震荡，适合网格型操作
  ├── HIGH_VOLATILITY  → 高波动，暂停交易
  └── LOW_VOLATILITY   → 低波动，等待机会

模型：LightGBM 分类器
训练：每 4 小时用最近 30 天数据重新训练
```

### 3.2 方向预测器（Direction Predictor）

**做什么**：预测未来 4 小时内价格上涨还是下跌，涨多少跌多少。

```
输入特征（120+ 维）：
  ├── 价格特征：40 个技术指标（RSI、MACD、布林带、EMA等）
  ├── 成交量特征：OBV、成交量加权均价、量比
  ├── 市场微观结构：订单簿深度、买卖价差、成交量不平衡
  ├── 衍生品数据：资金费率、持仓量变化、多空比
  └── 时间特征：小时、星期几（加密市场有时间效应）

输出（回归）：
  ├── 预期收益率（4h 后）
  ├── 预测置信度（模型对自己预测的把握）
  └── 预期最大回撤（4h 内最坏情况）

模型：LightGBM 回归器 + XGBoost 回归器，取加权平均
训练：每 4 小时用最近 60 天数据重新训练
```

### 3.3 风险计算器（Risk Calculator）

**做什么**：根据账户状态和市场状态，计算本次交易的安全边界。

```
输入：
  ├── 当前账户总权益
  ├── 已有仓位（避免过度集中）
  ├── 市场状态（来自 Layer 1）
  ├── 预测置信度（来自 Layer 2）
  ├── 历史最大回撤容忍度（默认 15%）
  └── 交易所杠杆倍数限制

输出：
  ├── 最大仓位百分比（占账户权益的 x%）
  ├── 止损价格（基于 ATR 的 1.5 倍 + 关键支撑/阻力）
  ├── 止盈价格（基于波动率的动态目标）
  ├── 杠杆倍数（保守：2x-5x，激进时不超 5x）
  └── 是否允许交易（市场太危险就暂停）
```

### 3.4 决策仲裁器（Decision Arbitrator）

**做什么**：综合前三层输出，做最终买卖决定。

```
输入汇总：
  ├── Layer 1: 市场状态 = TRENDING_STRONG
  ├── Layer 2: 预期收益 = +1.2%, 置信度 = 0.75
  ├── Layer 3: 最大仓位 = 8%, 止损 = -1.5%, 止盈 = +2.5%

仲裁规则（硬编码安全边界，不可被 AI 覆盖）：

  1. 市场状态 = HIGH_VOLATILITY → 不平仓但禁止开新仓
  2. 置信度 < 0.55 → 不动
  3. 预期最大回撤 > 账户总权益 5% → 不动
  4. 已有亏损仓位 → 不开同方向新仓
  5. 资金费率为极端负值 → 只做多不做空
  6. 单日连续 3 笔亏损 → 停机 12 小时

如果通过所有安全检查 → 开仓（或加仓）
  输出：{ action, position_size, leverage, stop_loss, take_profit }
```

## 四、AI 自我优化循环

### 4.1 在线学习流程

```
每 4 小时:
  1. 下载最新 OHLCV + 订单簿数据
  2. 重新计算所有特征
  3. 用最新 30/60 天数据重新训练 Layer 1 + Layer 2
  4. 评估新模型：如果新模型在最近 7 天回测中夏普比 > 旧模型，则替换
  5. 记录模型版本和性能指标

每笔交易平仓后:
  1. 记录完整的交易日志（入场理由、出场理由、盈亏）
  2. 更新 Layer 3 的最大回撤容忍度
  3. 如果连续亏损，提高置信度阈值
```

### 4.2 回测验证

```
模型更新前必须通过回测验证：
  回测周期：最近 7 天
  必须指标：
    - 夏普比率 > 0.5
    - 最大回撤 < 15%
    - 胜率 > 40%
    - 盈亏比 > 1.5

如果新模型不达标 → 保留旧模型，记录失败原因
```

## 九、安全边界（硬编码，AI 不可修改）

| 规则 | 值 | 说明 |
|------|-----|------|
| 单笔最大仓位 | 20% 账户权益 | 永远不 all-in |
| 最大杠杆 | 5x | 保守杠杆 |
| 最大回撤停机 | 15% | 到线自动停，需手动重启 |
| 单日最大亏损 | 5% | 到线停机 12 小时 |
| 连胜降保 | 连续 3 笔亏损后仓位减半 | 防止连续亏损 |
| 极端行情 | 波动率突增 3σ 时暂停 | 黑天鹅保护 |
| 资金费率极端 | \|费率\| > 0.1% 时只顺势 | 防止被收高额费率 |
| 最小置信度 | 55% | 不够自信就不动 |

| 盈亏比 | > 1.5 | > 2.0 |
